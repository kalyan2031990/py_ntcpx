# py_ntcpx Output Documentation

## Overview

This document explains all output files generated by the py_ntcpx pipeline, organized by tier and purpose.

## Four-Tier NTCP Framework

### Tier 1: Legacy-A (QUANTEC LKB / RS, fixed)
- **Purpose**: Literature transportability validation
- **Models**: 
  - `NTCP_LKB_LogLogit_QUANTEC` - LKB Log-Logistic with QUANTEC parameters
  - `NTCP_LKB_Probit_QUANTEC` - LKB Probit with QUANTEC parameters
  - `NTCP_RS_Poisson_QUANTEC` - Relative Seriality Poisson with QUANTEC parameters
- **Output Files**: 
  - `code3_output/ntcp_results.xlsx` - Contains Tier 1 predictions
  - **Status**: Legacy, preserved for backward compatibility

### Tier 2: Legacy-B (MLE-refitted LKB / RS)
- **Purpose**: Radiobiology calibrated to cohort
- **Models**:
  - `NTCP_LKB_Probit_MLE` - MLE-fitted LKB Probit
  - `NTCP_LKB_LogLogit_MLE` - MLE-fitted LKB Log-Logistic
- **Output Files**:
  - `output/model_parameters_mle.json` - Fitted parameters (TD50, m, n, gamma50)
  - `output/tiered_ntcp_results.xlsx` - Contains Tier 2 predictions
- **Status**: New tier, appended to existing results

### Tier 3: Modern Classical (de Vette / CITOR style)
- **Purpose**: True clinical dose-response modeling
- **Models**:
  - `NTCP_LOGISTIC` - Multivariable logistic regression with DVH + clinical features
- **Features**: mean_dose, gEUD, V30, V50, D20, age, baseline_toxicity, smoking, chemotherapy
- **Output Files**:
  - `output/tiered_ntcp_results.xlsx` - Contains Tier 3 predictions
  - Bootstrap variable stability included in model training
- **Status**: New tier, appended to existing results

### Tier 4: AI (ANN, XGBoost, SHAP, uNTCP, CCS)
- **Purpose**: Maximum achievable predictability
- **Models**:
  - `ML_ANN` / `NTCP_ML_ANN` - Artificial Neural Network
  - `ML_XGBoost` / `NTCP_ML_XGBoost` - XGBoost gradient boosting
  - `uNTCP` - Uncertainty-Aware NTCP with confidence intervals
  - `CCS` - Cohort Consistency Score
- **Output Files**:
  - `code3_output/ntcp_results.xlsx` - Contains Tier 4 predictions
  - `code_shap_full/` - SHAP analysis outputs
- **Status**: Existing tier, preserved unchanged

## Output Files by Category

### Master Reports

#### `NTCP_4Tier_Master.xlsx`
Comprehensive Excel workbook with all tiers:
- **Sheet 1: Patient NTCPs** - All NTCP predictions for all tiers per patient-organ
- **Sheet 2: Model Parameters** - Parameters for QUANTEC, MLE, and Logistic models
- **Sheet 3: AUC_Brier_CCS** - Performance metrics (AUC, Brier score) and CCS for all models
- **Sheet 4: uNTCP Summary** - Uncertainty-aware NTCP with confidence intervals
- **Sheet 5: ML QA** - Machine learning quality assurance metrics
- **Sheet 6: Dose-Response Stats** - Statistical summary of dose-response relationships

**Status**: New file, replaces no existing files

### Tiered Analysis Outputs

#### `tiered_ntcp_results.xlsx`
Extended results with Tier 2 and Tier 3 predictions appended to existing Tier 1 and Tier 4 results.

**Status**: New file, extends `code3_output/ntcp_results.xlsx` without replacing it

#### `model_parameters_mle.json`
JSON file containing MLE-fitted parameters for Tier 2 models:
```json
{
  "Organ": {
    "LKB_Probit_MLE": {
      "TD50": value,
      "m": value,
      "n": value,
      "log_likelihood": value,
      "converged": true/false
    },
    "LKB_LogLogit_MLE": {
      "TD50": value,
      "gamma50": value,
      "log_likelihood": value,
      "converged": true/false
    }
  }
}
```

**Status**: New file

### Dose-Response Curves

#### `dose_response_tiers.png`
Combined dose-response curves for all tiers using gEUD:
- QUANTEC LKB (blue solid)
- MLE-LKB (green dashed, LOESS smoothed)
- Logistic NTCP (red dotted, LOESS smoothed)
- ANN (magenta dash-dot, LOESS smoothed)
- XGBoost (cyan dash-dot, LOESS smoothed)

#### `dose_response_{Organ}.png`
Organ-specific dose-response curves.

**Status**: New files, do not replace existing plots

### ML QA Validation

#### `ml_validation.xlsx`
Machine learning quality assurance metrics:
- Train vs Test AUC
- Overfitting gap
- Brier score
- Number of samples

**Status**: New file, non-disruptive addition

### CCS (Cohort Consistency Score)

#### CCS Columns in Results
- `CCS_QUANTEC` - CCS for QUANTEC models
- `CCS_MLE` - CCS for MLE-refitted models
- `CCS_Logistic` - CCS for logistic regression models
- `CCS_ML` - CCS for ML models (ANN, XGBoost)
- `CCS_Warning` - Warning level (none, low, medium, high, critical)
- `CCS_Safety` - Safety flag (OK, DO_NOT_USE)

**Status**: Extended from existing CCS implementation

## Redundancy Notes

### NTCP Predictions
- **Redundant but intentional**: Tier 1 predictions appear in both `code3_output/ntcp_results.xlsx` and `tiered_ntcp_results.xlsx`
  - **Reason**: Backward compatibility and tier comparison
  - **Action**: Use `tiered_ntcp_results.xlsx` for comprehensive analysis, `code3_output/ntcp_results.xlsx` for legacy workflows

### Model Parameters
- **Redundant but intentional**: QUANTEC parameters appear in multiple places
  - `code3_output/ntcp_results.xlsx` (Literature Parameters sheet)
  - `NTCP_4Tier_Master.xlsx` (Model Parameters sheet)
  - **Reason**: Different contexts (legacy vs. tiered framework)
  - **Action**: Use `NTCP_4Tier_Master.xlsx` for tiered analysis

### Performance Metrics
- **Redundant but intentional**: AUC and Brier scores calculated in multiple places
  - `code4_output/qa_summary_tables.xlsx` - Original QA metrics
  - `NTCP_4Tier_Master.xlsx` - Tiered framework metrics
  - `ml_validation.xlsx` - ML-specific QA
  - **Reason**: Different scopes (original vs. tiered vs. ML-specific)
  - **Action**: Use `NTCP_4Tier_Master.xlsx` for comprehensive tier comparison

## Scientific Interpretation

### Tier Comparison
1. **QUANTEC (Tier 1)**: Validates literature transportability - shows if published parameters work on your cohort
2. **MLE-LKB (Tier 2)**: Shows radiobiology calibrated to your cohort - reveals if your patients differ from literature
3. **Logistic NTCP (Tier 3)**: True clinical dose-response - captures multivariable relationships
4. **AI (Tier 4)**: Maximum predictability - shows best achievable performance

### CCS Interpretation
- **CCS > 0.95**: Patient within training cohort distribution - predictions reliable
- **CCS 0.80-0.95**: Minor deviation - use with caution
- **CCS 0.50-0.80**: Moderate deviation - verify clinical parameters
- **CCS 0.20-0.50**: Significant deviation - predictions may be unreliable
- **CCS < 0.20**: Out-of-distribution - DO NOT use for clinical decisions

### uNTCP Interpretation
- **CI width < 0.10**: Low uncertainty - reliable prediction
- **CI width 0.10-0.30**: Moderate uncertainty - use with caution
- **CI width > 0.30**: High uncertainty - prediction unreliable
