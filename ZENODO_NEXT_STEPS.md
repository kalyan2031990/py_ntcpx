✅ **Release Created!** Your GitHub release `v1.0.0` has been published.

Now follow these steps to get your NEW Zenodo DOI:

## Step 1: Enable Repository in Zenodo (5 minutes)

1. **Go to Zenodo GitHub Settings:**
   - https://zenodo.org/account/settings/github/
   - (Make sure you're logged in to Zenodo with your GitHub account)

2. **Enable Your Repository:**
   - Click **"Sync now"** button (to refresh repository list)
   - Find `kalyan2031990/py_ntcpx` in the list
   - Toggle the switch to **ON** (enable it)

3. **Grant Access (If Private Repository):**
   - If your repository is private, you may need to grant Zenodo access:
   - Go to: https://github.com/settings/applications
   - Find "Zenodo" in authorized OAuth apps
   - Make sure it has access to private repositories

## Step 2: Wait for Zenodo Processing (5-10 minutes)

- After enabling the repository, Zenodo will automatically detect your release
- It usually takes **5-10 minutes** for Zenodo to archive your release
- You'll receive an **email notification** when it's ready
- Or you can check manually at: https://zenodo.org/deposit

## Step 3: Get Your NEW DOI

1. **Check Zenodo Dashboard:**
   - Go to: https://zenodo.org/deposit
   - Look for your archived release (should show "py_ntcpx v1.0.0")

2. **Copy the DOI:**
   - The DOI will be in format: `10.5281/zenodo.XXXXXXXX`
   - Example: `10.5281/zenodo.12345678`
   - Copy this DOI

3. **Verify:**
   - Click on the DOI link to verify it works
   - The page should show your archived release

## Troubleshooting

- **Repository not showing in Zenodo?** 
  - Make sure you clicked "Sync now"
  - Check that the repository name matches exactly

- **Release not archived after 10 minutes?**
  - Make sure the repository toggle is ON
  - Check your email for error notifications
  - Try creating another release or contact Zenodo support

- **Private repository access?**
  - Make sure Zenodo has access in GitHub OAuth settings
  - The archived version on Zenodo will be public (but your GitHub repo stays private)
